Medical Image Segmentation — U-Net (PyTorch)
Apr 2026
PyTorch U-Net NIfTI Medical AI Dice >0.82

Implemented the U-Net architecture from scratch in PyTorch for hippocampus segmentation on brain MRI scans — the same clinical task that powers organ-at-risk delineation in radiation oncology software. The hippocampus occupies less than 2% of the MRI volume, making standard loss functions useless; a combined DiceBCE loss handles the extreme class imbalance directly. Built the full pipeline: NIfTI ingestion, z-score normalisation, 2D slice extraction, data augmentation, training with cosine annealing and gradient clipping, and evaluation with Dice, IoU, precision, and recall. Validated on the Medical Segmentation Decathlon Task04 benchmark.

  • ~31 million parameter U-Net: 4-level encoder (64→128→256→512 channels), symmetric decoder, skip connections via concatenation — not addition — to preserve fine boundary detail
  • Combined DiceBCE loss: Dice handles <2% foreground class imbalance, BCE stabilises early-training gradients — standard in clinical segmentation pipelines
  • Complete ETL pipeline: load NIfTI volumes with nibabel, z-score normalise, resize slices to 256×256, filter near-empty slices, augment, save as .npy for fast DataLoader access
  • Cosine annealing LR schedule, gradient clipping at norm=1.0, best-checkpoint tracking by validation Dice — production training practices throughout

Breast Cancer Histopathology Classification
Apr 2026
ResNet50 Transfer Learning FastAPI PyTorch 0.962 sensitivity

A full histopathology classification pipeline for benign vs malignant patch detection using the BreakHis dataset. Built with patient-level train/test splitting to prevent data leakage, augmentation-heavy training on ResNet50 with transfer learning, and clinical evaluation metrics relevant to oncology workflows. Deployed as a FastAPI inference service connected to the trained checkpoint for local serving.

  • Patient-level splitting ensures no patient's patches appear in both train and test sets — critical for unbiased clinical evaluation and a common failure mode in naive histopathology pipelines
  • ResNet50 fine-tuned via transfer learning: frozen backbone in early epochs, progressive unfreezing — standard practice for small medical datasets with limited labelled examples
  • FastAPI inference service wraps the trained checkpoint, accepting JPEG patches and returning benign/malignant predictions with confidence scores for local clinical serving
  • Explored multimodal extensions: synthetic clinical feature generation, synthetic segmentation masks, and attention-based fusion components for future work

Real-Time E-Commerce Analytics Platform
Apr 2026
Kafka Spark Hadoop Airflow dbt Docker PostgreSQL

A fully distributed, production-grade data platform that processes live e-commerce events in real time. Three Python producers simulate an actual online store — orders, user clicks, and inventory movements — pushing events into a 3-broker Kafka cluster at 65+ messages per second. Spark Structured Streaming consumes all three topics simultaneously, validates schemas, deduplicates using watermarking, and writes Snappy-compressed Parquet to Hadoop HDFS every 5 minutes. Daily Spark batch jobs promote data through a full Medallion Architecture: Bronze (raw) → Silver (cleaned, type-cast, business-rule validated) → Gold (star schema with SCD Type 2 on customer and product dimensions). Three Airflow DAGs orchestrate everything on schedule. dbt builds the analytical marts and enforces 31 data quality tests automatically.

  • Kafka 3-broker cluster with replication factor 3 and min.insync.replicas 2 — no data loss if a broker goes down
  • Watermark-based deduplication in Spark Streaming handles late-arriving events up to 10 minutes without unbounded state
  • SCD Type 2 on dim_customers and dim_products — every past order joins to the dimension record that was current at the time of sale
  • 31/31 dbt tests passing: uniqueness, referential integrity, accepted values, no negative revenue, no duplicate current SCD records
